## 五、定时任务机制

### 1. Heartbeat（心跳轮询）

**原理：**
- 系统周期性轮询（如每 30 分钟）
- 检查 `HEARTBEAT.md` 中的任务清单
- 批量执行多项检查

**适合场景：**
- 邮件检查
- 日历检查
- 天气检查
- 项目状态检查

**配置：** 编辑 `HEARTBEAT.md` 文件

### 2. Cron（定时任务）

**原理：**
- 精确时间点触发
- 生成独立的子会话执行任务
- 结果直接投递到指定渠道

**适合场景：**
- 每天早上固定时间推送信息
- 一次性提醒
- 需要精确时间的任务

**配置方式（CLI）：**
```bash
openclaw cron add     # 添加定时任务
openclaw cron list    # 列出任务
openclaw cron remove  # 删除任务
```

或通过配置文件：
```yaml
# ~/.openclaw/config
cron:
  - name: "morning-job-alert"
    schedule: "0 10 * * *"  # 每天10点
    task: "搜索今日招聘信息"
    channel: "feishu:群ID"
```

## 七、总结关系

- **Skills（27个）** = 方法论/工具包
- **Function Call（55+个）** = 结构化 API（包含技能提供的飞书工具）
- **CLI（13+个）** = 命令行工具（通过 `exec` 调用）

**Function Call vs CLI 的区别：**
- **Function Call**：结构化调用，返回 JSON，适合程序化操作
- **CLI**：执行命令，返回 stdout/stderr，适合复杂脚本
